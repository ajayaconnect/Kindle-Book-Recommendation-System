01 — Project Overview
Making the World of Books Navigable

With millions of books across genres, readers often face discovery paralysis. This project builds a multi-strategy recommendation engine modelled on how platforms like Amazon and Netflix serve personalised content. Three complementary approaches — Collaborative Filtering, Content-Based (TF-IDF), and k-NN — were implemented and evaluated. The best-performing model was serialised and deployed via a Streamlit web application for real-time, interactive book recommendations.

03 — ML Pipeline
End-to-End Workflow
01
Data Loading & Merging
Loaded Books, Users, and Ratings CSVs. Merged on ISBN and User-ID. Dropped irrelevant image URL columns. Standardised column names across all three tables.
02
Data Cleaning & Filtering
Custom function replace_df_value fixed mismatched data. Converted Year column from object to integer. Filtered to users with ≥200 ratings and books with ≥50 ratings to reduce sparsity.
03
EDA & Visualisation (15 Charts)
Countplots, histograms, boxplots, bar charts, line plots, heatmaps, and pair plots. Key findings: Nora Roberts is the most prolific author, Ballantine Books leads publishers, Wild Animus has the most ratings.
04
Hypothesis Testing (3 T-tests)
Tested: USA vs Canada average user age; age of high vs low raters; number of ratings for high vs low rated books. T-tests selected for comparing means of two independent groups.
05
NLP Text Preprocessing
Applied on book title and review text for Content-Based Filtering:
Contraction Expand→ Remove Punctuation→ Remove URLs/Digits→ Remove Stopwords→ Lemmatization→ TF-IDF Vectorizer
06
Model Building & Evaluation
Three recommendation strategies built and evaluated on Recall@5, Recall@10, and Hit Rate. 80/20 train-test split. Best model selected for deployment.
07
Serialisation & Deployment
Best model saved using Joblib/Pickle. FastAPI REST endpoint built. Streamlit app deployed on Streamlit Cloud for live interactive recommendations.
04 — Model Comparison
Three Recommendation Strategies
Model 01
Collaborative Filtering
95.76%
Recall@5 & Recall@10
Leverages collective user behaviour — users with similar reading/rating patterns get similar recommendations. Exceptional recall across top-5 and top-10 slots.
Model 02
Content-Based Filtering
~0.15%
Recall@5 & Recall@10
TF-IDF + cosine similarity on book content. User profiles built by averaging item vectors weighted by interaction strength. Low recall due to sparse content overlap.
Model 03
k-NN (Cosine)
Hit Rate
Primary Metric
k-Nearest Neighbours with cosine distance in high-dimensional feature space. Measures whether at least one relevant item appears in top-k. Strong for book-to-book similarity.
Why Collaborative Filtering Wins

"Collaborative Filtering demonstrated exceptional performance with an impressive recall rate of 95%. This means that for nearly all users, the model successfully included at least one relevant book in the top-10 recommendations — the highest bar for a recommendation system and directly tied to user satisfaction and session depth."
